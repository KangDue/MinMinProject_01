7. 요소
요소(element)
- Html의 요소는 시작태그와 종료 태그 그리고 태그 사이에 위치한 내용으로 구성된다.
- Tag로 컨텐츠를 감싸는 것으로 그 정보의 성격과 의미를 정의
- 내용이 없는(self closed) 태그도 있다. (br, hr, img, input, link, meta)
- 요소는 중첩(nested, 포함)될 수 있음
- 요소의 중첩을 통해 하나의 문서를 구조화 할 수 있다.
- 여는 태그와 닫는 태그의 쌍을 잘 확인해야한다.(진짜로)
0회째 보는 중.
8. 속성
속성(Attribute)
- 속성을 통해 태그의 부가적인 정보를 설정할 수 있음
- 요소는 속성을 가질 수 있으며, 경로나 크기와 같은 추가 정보 제공
- 요소의 시작 태그에 작성하며 보통 이름과 값이 하나의 쌍으로 존재
- 태그와 상관없이 사용 가능한 속성들도 있음(HTML Global Attribute)
- 그 외에는 태그별로 속성이 다르다.
- 글로벌 속성이라도 아무 효과 없는 경우도 있으니 확인하고 사용
속성명 설명
id 문서 전체에서 유일한 고유 식별자 지정(고유하지 않아도 작동은 함)
class 공백으로 구분된 해당 요소의 클래스의 목록 (CSS, JS에서 요소를 선택하거나 접근)
data-* 페이지에 개인 사용자 정의 데이터를 저장하기 위해 사용
*은 사용자가 아무 이름 붙이면 됨. ex) data-level="1" 이런 식
style inline 스타일 정의
title 요소에 대한 추가적인 정보 지정
tabindex 요소의 tab index로 화면에서 tab을 누르면 몇번째로 선택되는지 정함.

Form details
속성명 설명
action 양식 데이터를 처리할 프로그램의(or 백엔드 서버) url/파일

을 현재 창이 아닌 다른 위치에서 열도록 지정

_self: 기본값, 현재 프레임, _top:윈도우 전체

15. Input 파고들기
form이 전체양식, 화면에 보이지 않는 추상적인 녀석임에 비해
input은 실제로 사용자가 양식을 입력하기 위한 태그이다.
type = 종류, name = 이름 (id 권장), value = 기본값
속성명 설명
type
  • text: 일반 텍스트
  • password: 비밀번호
  • button: 버튼
  • submit: 양식 제출용 버튼
  • reset: 양식 초기화용 버튼
  • radio: 한개만 선택 가능 구성요소
  • checkbox: 다수 선택 가능 구성요소
  • file: 파일 업로드
  • hidden: 사용자에게 안보이는 숨은값(default value)
name input의 이름지정(id 권장)
id input의 이름
readonly 읽기 전용으로 변경
maxlength 최대 글자수 지정
required 필수 입력으로 지정, 입력안하고
submit 하면 에러메시지 출력
autofocus 웹 페이지가 로딩되자마자
이 속성을 지정한 태그로 포커스 이동
palceholder 태그에 입력할 값에 대한 힌트(회색 투명값)
pattern 정규표현식을 사용하여 특정 범위 유효값만 받기
기타 input의 속성은 아니지만 textarea tag는
여러줄의 입력을 받을 수 있다.
autocolplete 속성은 자동완성 기능

17. CSS

Cascading Style Sheet

스타일을 지원하기 위한 언어
- 선택하고(선택자)
- 스타일을 지정(선언)한다.
(속성(property) : 값(Value);)
- 선택자{ 속성1:값1; ...} 형식

정의방법
  • 인라인(inline)
    해당 태그에 직접 style 속성 활용
    중복&찾기 어려움 -> 실수 잦아짐
  • 내부 참조(Embeding) - <style>
    head 내에 style에 지정
    코드가 너무 길어짐
  • 외부 참조(link file) - 분리된 css 파일, 링크
    외부 파일을 link를 통해 불러옴
    가장 많이 씀.

,를 기준으로 나열한다.
tag1, tag2, .class1, #id1 {}

상위 하위 요소를 정의해서 선택하려면
1.자식결합자 상위>하위 {}
2.자손결합자 상위 하위같이 쓰면 된다.
이는 형제,부모 선택자 등에도 함께 적용된다.
1의 경우 직계만 탐색하고, 2의 경우는 자손의 자손까지 탐색한다.
다만 스타일에 따라 상속이 되기도 하니 이점 유의할 것!
클래스 선택자 마침표(.)문자로 시작하며, 해당 클래스명 선택
.class명 {}
아이디 선택자 #문자로 시작하며, 해당 아이디명 선택
일반적으로 하나의 문서에 고유 값 하나 설정
여러개여도 동작은 함.(단일 사용 권장)
#id명 {}
형제 선택자
  1. 인접 형제 선택자
    선택자1 + 선택자2 {}
    선택자1 바로 뒤 선택자2를 선택한다
    1. 가장먼저 body 내부 최상위 태그 탐색
    2. 하면서 각 태그별 nested 태그 순서대로 탐색
  2. 일반 형제 선택자
    선택자1 ~ 선택자2 {}
    같은 계층에서 선택자1 뒤에 있는
    모든 선택자2를 선택한다.
부모 선택자 jquery의 $(요소).parent 같은 방법을 써야함.
속성 선택자 요소["속성명"] 또는 요소["속성명"="값"]
위 처럼 사용한다. 해당 속성에 해당하는 요소만 선택 가능.
요소:nth-child(n) 각 계층별 요소의 n번째 자식만 고른다.
카운트는 모든 요소를 동등하게 보고 셈
n으로 만든 수식 ok, odd, even 등등 ok(1부터)
nth-last-child라고 거꾸로 시작하는 것도 있다.(0부터)
요소:nth-of-type(n) 각 계층별 요소의 n번째 자식만 고른다.
카운트는 해당 type만 보고 셈
n으로 만든 수식 ok, odd, even 등등 ok(1부터)
nth-last-of-type라고 거꾸로 시작하는 것도 있다.(0부터)

태그명 설명
<blockquote>
(=text contents)		 텍스트가 긴 인용문을 나타낸다.
주로 들여쓰기를 한 형태로 그려진다.
인용문의 출처 url이나 text는 cite 요소로 제공 가능.
<pre>
(=text contents)		 미리 서식을 지정한 텍스트를 의미
HTML에 작성한 내용 그대로 표현된다.
<div>
(=text contents)		 플로우 콘텐츠를 위한 통용 컨테이너(기본 양식)
css로 꾸미기 전에는 콘텐츠와 레이아웃에 어떤 영향도 주지 않는다.
<hr>
(=text contents)		 이야기 장면 전환, 구획 내 주제 변경 등,
문단 레벨 요소에서 주제의 분리 (= 분리선)
<ol>
(=text contents)		 정렬된 목록을 나타내며, 보통 숫자 목록으로 표현
주로 메뉴, 목차, 색인등에 활용됨.
<ulr>
(=text contents)		 정렬되지 않은 목록을 나타내며, 보통 불릿으로 표현
<li>
(=text contents)		 하나의 문단을 나타낸다.
기타 태그들
(=text contents)		 menu, figure, dt,di, dd 등이 있다.
